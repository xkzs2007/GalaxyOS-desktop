## What this proves

The **whole desktop app actually runs and renders**:
- Sidecar starts on :5757 (zmq REP) and :5758 (HTTP SSE) — `uptime_s: 3.84`
- Renderer loads from `desktop-shell/renderer/index.html` (ZCode 3-column layout)
- `@jboltai/tokui` UMD bundle loads from `https://unpkg.com/@jboltai/tokui/dist/tokui.umd.js`
- Welcome bubble renders through the real TokUI client (AI avatar, Assistant + GalaxyOS badge, markdown with headings/code/lists, action chips)
- `/sse/process` streams 14 TokUI DSL fragments (bubble → think-chain → 2 tool-calls → markdown answer → confidence → actions → close)
- `connection: close` header replaces buggy `connection: keep-alive` + `content-length: 0` (which had been causing the browser's fetch() to see an empty body)

## Why this matters

This is **the first visual proof** that the entire stack works:

1. ✅ **OpenClaw decoupled** — shim replaces path_resolver; no OpenClaw needed
2. ✅ **Sidecar SSE** — stdlib asyncio HTTP server, 14 DSL fragments per process()
3. ✅ **TokUI integration** — real @jboltai/tokui UMD bundle rendering DSL into styled DOM
4. ✅ **ZCode/Codex layout** — 3-column grid (sidebar / center / details)
5. ✅ **Streaming** — fetch() reads chunks; ui.feed() appends DOM; ui.endStream() closes
6. ✅ **Engine works** — `XiaoYiClawLLM.process()` returns a real R-CCAM 5-phase trace (graceful degradation for missing submodules)

## How to reproduce

```bash
# 1. Start the sidecar
python -c "
import sys; sys.path.insert(0, 'desktop-shell/python')
import asyncio
from galaxyos_sidecar import main_async
asyncio.run(main_async())
"

# 2. Serve the renderer
cd desktop-shell/renderer && python -m http.server 8080

# 3. Run the Playwright smoke test
python desktop-shell/scripts/playwright_smoke.py

# → Outputs to C:\Users\Administrator\Desktop\galaxyos-desktop-{initial,after-ask}.png
```

For production Electron, replace step 2 with `npm run dev` (which spawns the sidecar, builds with esbuild, and opens the Electron window).
