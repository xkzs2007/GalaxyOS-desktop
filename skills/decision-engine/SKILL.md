# 决策引擎

> "用结构化方法分析决策，权衡利弊，量化风险。"

## 核心理念

好的决策不是"拍脑袋"，而是：
- 列出所有选项
- 定义评估维度
- 量化打分
- 权衡风险

## 什么时候使用

- 用户说"帮我做个决策"
- 面临多个选项需要选择
- 需要权衡利弊
- 重大决策需要理性分析

## 工作流程

```
决策问题 → 列出选项 → 定义维度 → 打分评估 → 推荐方案
```

### 步骤详解

**1. 定义决策问题**
- 要决定什么？
- 截止时间？
- 决策标准？

**2. 列出选项**
- 所有可能的选择
- 包括"不行动"选项

**3. 定义评估维度**
- 哪些因素重要？
- 各占多少权重？

**4. 打分评估**
- 每个选项在每个维度打分
- 计算加权总分

**5. 推荐方案**
- 基于分数推荐
- 说明理由和风险

## 决策工具

### SWOT分析
- Strengths（优势）
- Weaknesses（劣势）
- Opportunities（机会）
- Threats（威胁）

### 决策树
```
        决策点
       /      \
    选项A     选项B
    /  \      /  \
  成功 失败  成功 失败
  70%  30%  50%  50%
```

### 预期值计算
```
预期值 = Σ (结果价值 × 概率)
```

## 注意事项

1. **避免分析瘫痪**
   - 不是所有决策都需要详细分析
   - 小决策快速决定

2. **承认不确定性**
   - 很多因素无法精确量化
   - 用区间而非精确值

3. **考虑机会成本**
   - 选择A意味着放弃B
   - 要考虑放弃的价值

4. **定期复盘**
   - 决策后跟踪结果
   - 学习改进决策方法
