# 健康概览 CLI 获取与分析指南

## 一、数据获取策略

### 今日健康概览

```bash
node ./skills/xiaoyi-health/bin/pha-claw.js get_activity_data --date today
```

### 本周趋势

```bash
node ./skills/xiaoyi-health/bin/pha-claw.js get_activity_data --last-days 7
```

### 指定日期范围

```bash
node ./skills/xiaoyi-health/bin/pha-claw.js get_activity_data --start-date 2024-01-01 --end-date 2024-01-07
```

## 返回字段说明

| 字段 | 说明 |
|------|------|
| `steps` | 当日步数 |
| `calories` | 消耗卡路里（千卡） |
| `activeMinutes` | 中高强度活动时长（分钟） |
| `activeHours` | 活动小时数 |
| `stepsGoal` | 步数目标 |
| `activeCaloriesGoal` | 活动热量目标 |
| `exerciseTimeGoal` | 锻炼时长目标（分钟） |

## 二、分析框架

### 确定查询范围

| 用户描述 | 范围 | 数据策略 |
|---------|------|---------|
| "我怎么样？" / "健康概览" | 全面概览 | `get_activity_data(today)` |
| "今天走了多少步？" | 单项指标 | `get_activity_data(today)` |
| "这周怎么样？" | 周趋势 | `get_activity_data(last_days=7)` |
| "达标了吗？" | 目标追踪 | `get_activity_data` + 检查个人目标 |
| "这周和上周比" | 周期对比 | 两段日期范围分别获取 |

### 步数分级

始终**先与用户自己的目标和历史对比**，再参考人群基准：

| 步数 | 活动水平 |
|------|---------|
| < 4,000 | 久坐 |
| 4,000–7,999 | 低活动量 |
| 8,000–9,999 | 一般活动量 |
| 10,000–12,499 | 活跃 |
| > 12,500 | 高度活跃 |

**活动时长比步数更能说明质量**：慢走 10,000 步 ≠ 包含 30 分钟慢跑的 6,000 步。

### 久坐风险指标

| 指标 | 目标 | 风险阈值 |
|------|------|---------|
| 连续久坐时间 | 每次 < 1 小时 | > 2 小时不间断 |
| 中等以上强度活动 | 每周 ≥ 150 分钟 | < 75 分钟/周 |

**NEAT（非运动性活动产热）**：久坐与活跃者的差异可达每天 500+ 千卡，往往是体重管理的隐藏变量。

**打破久坐策略**：每小时站立提醒、接电话时走动、走楼梯代替电梯。

### 目标进度沟通

| 进度 | 语气 |
|------|------|
| > 100% | 庆祝，肯定努力 |
| 80–100% | 鼓励，差一点点 |
| 50–80% | 中性 + 建议，还有时间 |
| < 50% | 共情，了解原因 |

**目标调整信号**：
- 连续 2 周以上超出目标 > 30% → 建议提高目标
- 连续 2 周以上低于目标 60% → 讨论目标是否需要调整

### 周度趋势分析

- **规律性**比总量更重要：5 天达标比 1 天走 15,000 步更有意义
- 趋势方向：逐周改善→肯定；逐周下降→温和了解原因
- 连续 2 周以上低于目标 60% → 讨论目标调整

### 卡路里注意事项

穿戴设备卡路里数据误差显著（±20–30%），应讨论**相对变化**而非绝对数字，不用于指导饮食行为。

### 跨域关联

| 观察 | 关联 | 如何呈现 |
|------|------|---------|
| 步数少 + 昨晚睡眠差 | 睡眠影响动力 | "你昨晚只睡了 5.5 小时，这可能解释了今天比较安静。休息和恢复同样重要。" |
| 步数多 + 睡眠好 | 良性循环 | 活跃的一天加上良好睡眠，肯定正向循环 |
| 连续多天步数骤降 | 需要了解情况 | 温和关心，可能生病/受伤/出差 |

### 数据总结规范

当用户明确要求"总结今天所有数据"、"用数字告诉我"时，完整展示所有数据点：

```
| 指标          | 数值        | 状态              |
|--------------|------------|-------------------|
| 🚶 步数       | 8,200 步   | 目标 10,000 的 82% |
| 🔥 活跃卡路里  | 420 千卡   | 高于日均           |
| ⏱️ 活动时长   | 28 分钟    | 达标 ✅            |
```

数据展示完毕后给一句总结性洞察。

### 红线

| 信号 | 处理方式 |
|------|---------|
| 连续多天活动量骤降至接近零 | 温和关心："我注意到你最近几天活动量很低。一切都好吗？" |
| 用户明显生病/受伤但仍问目标进度 | 优先恢复："你的身体现在需要休息，目标在你好了以后还在那里。" |
| 用户过度关注每日数字 | 引导看趋势："每天的波动完全正常，真正重要的是周趋势。" |

## 输出格式要求

- **禁止 Markdown 绘图**：不得输出任何图表语法，包括 Mermaid 图、折线图、柱状图、饼图，以及用字符拼凑的 ASCII 图表
- 合理使用标题（`##`）、列表、**加粗**、表格等排版方式呈现数据和分析结果
