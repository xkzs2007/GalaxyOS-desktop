# 睡眠数据获取与分析指南

## 日期偏移规则

睡眠数据记录在**醒来日期**下，查询时必须做日期偏移：

| 用户表述 | 应查询的日期 | 说明 |
|---------|-----------|------|
| "昨天 / 昨晚 / 今天的睡眠" | 今天 | 最近一次醒来（今天早上） |
| "前天的睡眠" | 前天 | 前天醒来 |
| "前天晚上的睡眠" | 昨天 | 前天入睡 → 昨天醒来（日期 +1） |
| "X 日晚上的睡眠" | X+1 日 | 当晚入睡，次日醒来（日期 +1） |
| "今晚的睡眠" | 不传时间参数 | 尚未发生 |

**核心规则**：表述含"晚上" → 日期 +1；不含"晚上" → 直接使用该日期。

**⚠️ 重要**：查询"昨天 / 昨晚 / 今天的睡眠"时，必须使用 `"date": "today"` 而非推算具体日期（如 `"2026-03-28"`）。`today` 由服务端实时解析，避免模型日期判断偏差。

**具体示例（假设今天是 3 月 27 日）**：

- "昨天睡得怎么样" → 查 **3 月 27 日**（昨晚入睡，今早醒来，记录在今天）
- "昨晚睡得怎么样" → 查 **3 月 27 日**（同上，含"晚"但指昨夜→今晨，仍记录在今天）
- "今天的睡眠" → 查 **3 月 27 日**（今早刚醒，数据记在今天）
- "前天睡得怎么样" → 查 **3 月 25 日**（前天醒来，记在前天）
- "前天晚上的睡眠" → 查 **3 月 26 日**（前天夜间入睡 → 昨天醒来，日期 +1，记在 26 日）
- "3 月 25 日晚上的睡眠" → 查 **3 月 26 日**（25 日夜间入睡 → 26 日醒来）
- "今晚的睡眠" → **不传时间参数**（尚未发生）

**⚠️ 无数据时严禁替换日期**：若查询返回 `null`，如实告知"该日期暂无睡眠记录"，**绝不可用前一天或其他日期的数据来回答**。用户问的是昨晚的睡眠，即使前天有数据也不能用来代替作答。

## 一、数据获取策略

### ⚠️ 无数据处理规则

查询日期无睡眠记录时，接口直接返回 `null`，**不会自动回溯到其他日期**。

| 情况 | 处理方式 |
|------|---------|
| `data` 为 `null` | 告知"该日期暂无睡眠记录，可能未同步或未佩戴手表" |
| 有数据 | `date` 字段即为查询日期，正常分析 |

### 场景 A：单日睡眠分析

```bash
# 基础数据（每次必取）
node ./skills/xiaoyi-health/bin/pha-claw.js get_sleep --date today

# 多指标并行获取（推荐，效率更高）
node ./skills/xiaoyi-health/bin/pha-claw.js --tools '[
  {"name":"get_sleep","args":{"date":"today"}},
  {"name":"get_heart_rate","args":{"date":"today"}},
  {"name":"get_stress","args":{"date":"today"}},
  {"name":"get_workouts","args":{"date":"today"}}
]'
```

### 场景 B：睡眠趋势分析（近 7 天）

```bash
node ./skills/xiaoyi-health/bin/pha-claw.js --tools '[
  {"name":"get_sleep","args":{"startDate":"YYYY-MM-DD","endDate":"YYYY-MM-DD"}},
  {"name":"get_heart_rate","args":{"startDate":"YYYY-MM-DD","endDate":"YYYY-MM-DD"}},
  {"name":"get_stress","args":{"startDate":"YYYY-MM-DD","endDate":"YYYY-MM-DD"}}
]'
```

或使用便捷参数：
```bash
node ./skills/xiaoyi-health/bin/pha-claw.js get_sleep --last-days 7
node ./skills/xiaoyi-health/bin/pha-claw.js get_heart_rate --last-days 7
```

### 场景 C：夜间频繁清醒（排查呼吸暂停）

```bash
node ./skills/xiaoyi-health/bin/pha-claw.js --tools '[
  {"name":"get_sleep","args":{"date":"today"}},
  {"name":"get_spo2","args":{"date":"today"}},
  {"name":"get_heart_rate","args":{"date":"today"}}
]'
```

### 关联数据采集决策表

| 用户问题 | 额外取数 | 排查逻辑 |
|---------|---------|---------|
| 睡不着 / 入睡慢 | `get_workouts` `get_heart_rate` `get_stress` | 睡前高强度运动？就寝时心率偏高？压力 ≥60？ |
| 半夜老醒 | `get_heart_rate` `get_spo2` `get_stress` | 夜间心率突升？SpO2 反复 <95%（呼吸暂停）？ |
| 醒得太早 | `get_stress` | 压力或情绪异常？ |
| 睡了还是累 | `get_workouts` `get_heart_rate` | 深睡占比低？运动负荷过大？ |

## 二、分析框架

### 第一步：分类睡眠问题

| 用户描述 | 问题类型 | 主要调查方向 |
|---------|---------|------------|
| "睡不着" / "入睡太慢" | **入睡困难型** | 就寝时间规律性、睡前行为 |
| "半夜老醒" | **维持困难型** | 清醒次数、深睡分布、夜间心率 |
| "醒得太早" | **早醒型** | 总时长、REM 时段、压力指标 |
| "睡了很久还是累" | **非恢复性睡眠** | 深睡占比、质量评分 |
| "睡太多 / 起不来" | **嗜睡** | 睡眠质量、活动水平 |
| "我的睡眠怎么样？" | **概览请求** | 拉取 7 天趋势，与个人基线对比 |

### 第二步：时长评估

人群参考值：
- 成人（18–64 岁）：6–10 小时
- 老年人（65 岁+）：6–9 小时

**规律性比绝对时长更重要**：查看一周就寝/起床时间，计算波动幅度：
- 波动 < 60 分钟：优秀
- 波动 60-90 分钟：可接受
- 波动 > 90 分钟：昼夜节律紊乱，优先改善规律性

### 第三步：睡眠结构分析

| 阶段 | 目标范围 | 功能 | 影响因素 |
|------|---------|------|---------|
| **深睡** | 20-60% | 身体恢复、免疫、生长激素 | 运动时间、酒精、年龄 |
| **浅睡** | < 55% | 过渡阶段；比例过高 = 深睡不足 | 通常填充剩余时间 |
| **REM** | 10-30% | 记忆巩固、情绪调节 | 睡眠时段、酒精、压力 |
| **清醒** | ≤ 1 次且 ≤ 10 分钟 | 正常微觉醒 | 噪音、温度、疼痛 |

**深睡偏低时检查**：睡前运动、饮酒、就寝过晚
**REM 偏低时检查**：过早醒来（削减 REM）、酒精（显著抑制 REM）

### 第四步：睡眠障碍指标

| 模式 | 可能问题 | 建议 |
|------|---------|------|
| 频繁醒来（>3 次）+ SpO2 <95% | 阻塞性睡眠呼吸暂停 | 建议正式睡眠检查 |
| 睡眠心率未低于日间心率 | 交感神经过激 | 调查压力、过训、咖啡因 |
| 周末比工作日多睡 2 小时以上 | 社交时差 | 优先保持每天一致的起床时间 |
| 过早觉醒 | 抑郁情绪、皮质醇异常 | 关注情绪状态 |

### 第五步：跨域分析

**睡眠 + 运动**：适度运动改善睡眠，但睡前 2 小时内高强度运动可能延迟入睡
**睡眠 + 心率**：睡眠期间心率应低于日间；夜间心率升高提示压力/生病/过训
**睡眠 + 压力**：压力评分 ≥60 时，交感神经未切换到放松模式，影响入睡和深睡

### 第六步：个性化建议

每条建议必须引用用户的具体数据（禁止笼统建议）：

建议优先级：
1. **作息规律性** — 一切的基础
2. **时长** — 足够的总睡眠时间
3. **睡前程序** — 睡前一小时做了什么
4. **环境** — 温度、光线、噪音
5. **睡眠架构** — 深睡/REM 优化（进阶）

建议分层模板：
- **今晚立即执行**：具体的即时行动
- **本周调整**：运动时间、咖啡因、饮食时间等
- **长期优化**：固定起床时间（比固定就寝时间更重要）

### 红线

| 信号 | 行动 |
|------|------|
| 连续 7 天睡眠 < 5 小时 | 建议就医 |
| 用户提到呼吸暂停/严重打鼾 | 建议睡眠检查 |
| 持续性失眠（入睡 >30 分钟，每周多晚，持续 3 周以上） | 推荐 CBT-I 认知行为疗法 |

### 注意事项

- 穿戴设备睡眠阶段识别准确性不如专业多导睡眠监测（PSG），数据仅供参考
- 睡眠评分与个人历史均值相比，10% 以内的小幅下降属正常
- 分析应基于连续多天趋势，避免单日异常值误导

## 输出格式要求

- **禁止 Markdown 绘图**：不得输出任何图表语法，包括 Mermaid 图、折线图、柱状图、饼图，以及用字符拼凑的 ASCII 图表
- 合理使用标题（`##`）、列表、**加粗**、表格等排版方式呈现数据和分析结果
