# 系统思维

> "整体大于部分之和。" — 亚里士多德

## 核心理念

不要看孤立的要素，要看：
- **要素之间的关系**
- **反馈回路**
- **涌现现象**

## 什么时候使用

- 用户说"这是个系统性问题"
- 问题涉及多个相互影响的因素
- 需要理解整体行为
- 寻找杠杆点（改变哪里影响最大）

## 核心概念

### 1. 系统边界
- 系统包含什么？不包含什么？
- 边界决定了分析的范围

### 2. 要素与关系
- 要素：系统的组成部分
- 关系：要素之间的连接
  - (+) 正相关：A 增加 → B 增加
  - (-) 负相关：A 增加 → B 减少

### 3. 反馈回路
- **增强回路 (R)**：自我强化，导致指数增长或衰退
- **平衡回路 (B)**：自我调节，维持稳定

### 4. 涌现现象
- 系统整体表现出的新特性
- 无法从单个要素推导出来

### 5. 杠杆点
- 改变哪里能产生最大影响
- 通常在反馈回路的关键节点

## 经典案例

### 案例：公司增长停滞

**系统分析**：
- 要素：收入、研发投入、产品质量、市场份额
- 增强回路：收入 → 研发 → 产品质量 → 市场份额 → 收入
- 平衡回路：市场份额 → 竞争对手反应 → 价格战 → 利润率 → 收入

**杠杆点**：提高研发效率，打破平衡回路

## 注意事项

1. **不要过度简化**
   - 系统往往有多条反馈回路
   - 要考虑延迟效应

2. **识别关键回路**
   - 不是所有回路都同等重要
   - 找出主导行为的回路

3. **考虑时间延迟**
   - 因果之间往往有时间差
   - 延迟会导致震荡

4. **警惕意外后果**
   - 干预一个地方可能影响其他地方
   - 要考虑系统性影响
